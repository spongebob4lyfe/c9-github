How The Project Works Overall

Programming Software and Language

Overall, my "about Me and What I've Learned" page functions with a coding program that understands several programming languages. This allows the programmer to type up instructions and code that the computer can translate into a language that it understands. Basically, a coding, or programming, language is a series of syntax rules that can explain how code should be written and presented. The language a computer uses and understands is binary code, which is code made up of combinations of 0s and 1s that would take eons and an extremely advanced human mind to write out. Therefore, we use several different coding languages that we understand and can easily create, with certain languages made for certain purposes. When we write a certain command, it is first translated into assembly language, which is a low-level programming language for a computer, meaning it is closer to the binary language a computer understands, unlike the high-level language mostly used today. Then, the assembly language is translated into machine language, a set of instructions that is executed directly by the CPU, the computer's central processing unit, into binary code so that the computer can understand the code.

HTML

I used HTML, Hyper Text Markup Language, to place content related to myself on my "About Me" webpage, using several different tags to add various ideas and visuals. Simply put, HTML is a code that is used to write and construct each and every page on the Internet. It is constantly evolving and getting modified to meet the requirements of the world and its advances, so it is always improving. HTML consists of several short codes known as tags, which are known to be in between angle brackets (<,>). These tags allow us to easily put content into webpages and sites, since they are easy to understand and can be written pretty quickly. Each tag has a different, specific function, and the tags themselves don't appear on the page, but their effects on the content do.

CSS

I used CSS, Cascading Style Sheets, to design and "beautify" my "About Me" page, linking a CSS file to my HTML file so that I could add style to my page more quickly and easily from the CSS file, instead of just putting CSS elements into my HTML file, which would take forever and would make the structure of my page less appealing and sophisticated. CSS is basically a style language that sets up and formats the overall layout and design of the webpage. CSS relies on divs, class, id, and curly brackets in order to function correctly. It also has certain explicit terms that must be typed correctly in order to serve its purpose, which may be positioning, coloring, or changing the size of an item, depending on what the term is.

Code Specifics

HTML Tags

Headings

There are six sizes of headings, with six similar numbered heading tags shown in the image above. The 'h' stands for heading, followed by a number that defines its size. The lower the number is, the bigger the size, so the most important headings should be the largest, thus they would use a heading tag with a low number.

Paragraphs

The paragraph tag, shown in the image above, is for putting text into a webpage. The 'p' in the paragraph tag stands for paragraph, and between the opening and closing tags is where the content should be written.

Unordered Lists

This is an unordered list, which uses the tags on the left side of the image above and looks like the right side of the image. 'ul' stands for unordered list, and 'li' stands for list item. This means that each of the items are organized with bullet points, not numbers.

Ordered Lists

This is an ordered list, which uses the lags on the left side of the image above and looks like the right side of the image. 'ol' stands for ordered list, and 'li' stands for list item. This means that each of the items are numbered, instead of being organized with bullet points.

Nested Lists

This is a nested list. As you can see in the image above, there is an ordered list within another ordered list. One set of the ordered list tags is placed in between another st of ordered list tags, making it so that there is a numbered list within another numbered list. One set of list items is inside an original set of list items.

Images

To display an image on the webpage, an image tag is used. There is no closing tag, just a slash before the last angle bracket. First, the image address is placed in the quotations after 'src=' so that the image could be identified, like in the image above. Then, after 'alt=' a description of the image is written, mainly for those who cannot see the image, or if the image won't show, also shown in the picture above. Finally, although it is not shown above, there is a height and/or width attribute for controlling the size of an image. Ex.: height="200px"

Hyperlinks

Above is an image of a tag meant for displaying a hyperlink onto a webpage. In the opening tag, after the 'a', which stands for anchor, the link is placed in the quotes after 'href='. Then, after the opening tag, any name the link will be under must be written, followed by the closing tag.

Divs

The div tag is used to define a certain section of a webpage. It can group several different sections together, mainly for being styled with CSS. This tag is mostly used with CSS and accompanied with class and/or id.

Class and Id

Class and Id are often used with divs, used for applying an item or a group of items to one or more attributes. To identify certain classes and ids in CSS, they are named. Ex.: class="item1" However, although class is similar to id, they are still different. Class usually applies to several elements, while id is used when CSS is applied to one attribute only.

CSS Syntax

Linking CSS to HTML

The tag above shows how to link a CSS file to an HTML file, which allows for styling a webpage from a separate file, which makes CSS much faster and easier. First a CSS file must be created for the HTML file. The link tag doesn't have a closing tag, just a slash before the closing angle bracket. After 'link' is written, type 'rel="stylesheet" type="text/css" href="". This defines a CSS file, so in the quotations after 'href=', place in the name of the CSS file (cssfilename.css).

CSS Syntax

CSS syntax is the format used for styling in CSS. Like in the image above, there must first be the selector, which may be the whole 'body' or just a class or id name of a div. Basically, it is just anything that will be styled. After that is the declaration block, which is between two curly brackets and consists of the property and the value. The property is the part of the selector that will be modified, and the value is what it will be changed to. For example, the property can be 'background-color', and the value would be the color the background will be, like 'blue' or 'green'.

Class and Id

If an item has a class or id, then that means that it will probably be styled in CSS. The name of a class in CSS would have a '.' in front of it, while the name of an id would have a '#' in front of it. As it says above, class can be applied to several sections, while id can only be applied to one. Since class and id are usually in divs, if CSS is applied to a class or id, the sections grouped into the div that have a certain class or id will be affected by CSS and will be styled.

Colors

One of the most modified properties is color. Mostly everything on a webpage can have various colors, whether it is the background, borders, or the text itself. In addition, not only can background be a color, it can also be an image, as long as the URL of the image is put into the value section after 'url' and in between two parenthesis. Some examples of properties for color change are 'color:', 'background-color', or 'border-color'.

Padding, Borders, and Margins

Padding, Borders, and Margins are mainy used for positioning and looks. The size of padding and margins can be changed to create space between two items so that they are not all compact together, which isn't very appealing. They can also be used for positioning. For example, modifying the padding of an image inside a border can move it to the center, left, right, up , or down, depending on where the size of the padding is changed.
